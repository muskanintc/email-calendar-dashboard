Notes for Designer / Developer

  • One field only: email address. Fewer fields means more signups. Add name only if the client insists.
  • Trigger: desktop show after ~5 seconds or 40% scroll. Mobile on exit intent or after ~8 seconds. Do not fire on the cart or checkout pages.
  • Frequency: set a cookie so it does not reappear for a subscriber, and suppress for ~14 days after a dismiss.
  • Integration: submit adds the email to the ESP "Welcome Flow" list/segment. That enrollment triggers Welcome Emails 1 to 3. The 10% code is WELCOME10 (confirm exact code + any minimum/exclusions with client).
  • Success: swap the form for the success state above. Do NOT show the code on screen. Point them to their inbox and invite them to explore the site. The code arrives in Welcome Email 1.
  • Accessibility: real close button, focus trap inside the modal, escape to close, labelled email input.
  • Images used: roofer on a steep roof (image_11) fills the left image panel; boot on a shingle roof (22d116da) fills the dimmed page behind the popup. Both are live site images. Swap freely.
